# Текст для вставки в User Rules (все проекты)

Чтобы **все агенты во всех проектах** учитывали фазовую методологию, добавь в настройки глобальное правило.

## Что сделать

1. Открой **Settings** (Ctrl+,) → в боковой панели выбери **Rules** (или **Rules, Commands**).
2. В блоке **User Rules** нажми **Add Rule** или открой поле для правил.
3. Вставь текст из блока ниже в одно правило (например, название: «Methodology pack» или «Фазовая методология»).

---

## Текст правила (скопируй целиком)

```
Для задач, связанных с кодом и разработкой, читай и применяй правила фазовой методологии.

Путь к правилам: methodology-pack\rules\

В каждой сессии учитывай methodology-overview.mdc (порядок фаз, контекстная инженерия). Когда пользователь называет фазу или следует методологии — читай и применяй соответствующий файл: phase-research.mdc, phase-design.mdc, phase-planning.mdc, phase-implementation.mdc, phase-impl-code.mdc, phase-impl-tests.mdc, phase-impl-review.mdc, phase-review.mdc (Quality Gate).

Порядок фаз: Research → Design → Planning → Implementation (Code → Tests → Code Review) → Quality Gate. Не переходи к коду без дизайна и плана.
```

---

После сохранения это правило будет подставляться в контекст Agent (Chat) во всех проектах на этом компьютере. Если папку методологии перенёс — измени путь в правиле.
